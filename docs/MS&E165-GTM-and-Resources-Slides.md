## 5. Go-To-Market Strategy

### Slide: Launch Plan (Channels + Timeline)

**Title:** Launch Plan

**Channels**

| Channel                                         | Role                                                                                                                                                                  |
| ----------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **In-product (PearPad)**                  | Primary: PearNavigator ships as a system feature; first touch for 35M PearPad users, ~50% creatives. No separate install = maximum reach.                             |
| **Pear ecosystem (PearCloud, Pear Desk)** | Cross-sell: Promoted in creative workflows (e.g., “Use PearNavigator on PearPad for this task”). Drives multi-device engagement.                                    |
| **Creative community & education**        | Awareness: Short demos and “how I use PearNavigator” content with design influencers and bootcamps (e.g., Figma-focused). Builds credibility with Visual Designers. |
| **Enterprise / teams**                    | Expansion: Pitch to design leads and L&D for onboarding and reskilling; team adoption drives retention and upsell to PearCreate+.                                     |

**Timeline (align with your Execution Plan):** Alpha (50+ power users, Months 1–4) → Beta (broader rollout, Weeks 5–8) → Model tuning & automation (Months 5–6) → General availability (Month 6+).

### Slide: Target & Impact Metrics

**Title:** Target & Impact Metrics

One set of metrics: we set **targets** by phase and use the same metrics to **measure impact** post-launch (leading indicators → outcomes → business).

| Metric                                                               | Target (by phase)              | How we use it for impact                                                                     |
| -------------------------------------------------------------------- | ------------------------------ | -------------------------------------------------------------------------------------------- |
| **Suggestion acceptance**                                      | >85% (Alpha)                   | Validates guidance relevance; drives model/UX iteration.                                     |
| **2-week retention**                                           | >60% (Beta)                    | Confirms habit formation; signals product-market fit.                                        |
| **30-day adoption** (eligible creatives)                       | >20% (Launch)                  | Proves channel and positioning; we optimize in-product discovery from this.                  |
| **Task completion with Navigator**                             | Improve vs. baseline (ongoing) | % sessions where user completes intended task; measures guidance quality and drives roadmap. |
| **Abandonment to external help**                               | Reduce (ongoing)               | % sessions where user leaves for Google/videos; proves we cut context-switching.             |
| **Business** (PearPad engagement, NPS, PearCreate+ conversion) | Track (ongoing)                | Links product success to ecosystem and revenue.                                              |

---

*One-liner:* We review these weekly; targets define success by phase, and the same metrics feed iteration and impact reporting.

## 7. Required Resources

### Slide: Required Resources (Ask)

**Title:** Required Resources

| Resource | Rough scope / quantity | How it supports launch |
|----------|------------------------|-------------------------|
| **Funding** | ~$80K–120K through launch: cloud/API ~$15–25K; device lab / fleet testing ~$20–30K; alpha/beta incentives (50+ users) ~$5–10K; contingency ~$40–55K | Enables scale testing without cost caps, valid alpha/beta sample size, and runway for iteration before GA. |
| **Tools** | Vision/LLM APIs (or internal); OCR (e.g. Tesseract); device lab or cloud (PearPad + Figma, Photoshop, etc.); analytics (events, optional A/B) | Reliable inference + real-device parity; analytics feed adoption/execution/outcome metrics and roadmap. |
| **Team** | ~4–6 FTE: 1 Product, 2–3 Engineering (ML + overlay + infra), 1 Design, 0.5 Research (or part-time) | Clear ownership for build, quality, and user feedback; hit alpha/beta milestones and iterate on acceptance/retention. |
| **Partnerships** | 2–3 app/ecosystem (e.g. Figma, Adobe) for early access or co-marketing; 5+ community/bootcamp touchpoints for alpha recruitment and case studies | Credibility with Visual Designers; faster alpha recruitment; distribution and storytelling for post-launch and enterprise. |
